# Login Page Test Plan

## Objective

The goal of this test plan is to verify that the login functionality of the website works correctly and securely.

## Scope

This testing will focus on the login page and the ability for users to sign in using valid credentials.

## Features to Test

* Login with valid email and password
* Login with invalid password
* Login with invalid email
* Empty username or password fields
* Error messages for incorrect login

## Features Not in Scope

* User registration
* Password reset functionality

## Test Environment

Testing will be performed using:

* Desktop browser (Chrome)
* Desktop browser (Safari)

## Testing Approach

Manual testing will be used to verify that the login functionality works as expected.

## Risks

Possible risks include:

* Users being unable to log in
* Incorrect error messages being displayed

## Entry Criteria

The login page must be accessible in the browser.

## Exit Criteria

All planned test cases have been executed and results documented.
